Privacy Policy

Kern doesn't collect your data.
It can't — there's nothing to collect it with.

Effective 8 July 2026 · Applies to the Kern Chrome extension v1.0.0 and later
The short version: Kern runs entirely inside your browser. It has no server, no analytics, no account system and no telemetry. It does not collect, store remotely, transmit, sell or share any data about you or the pages you visit — not because a policy says so, but because there is no backend for data to go to.

Who we are

Kern is a free Chrome extension published by niallcodes, an independent software project operated by Florence Publishing Ltd, a company registered in the United Kingdom.

What Kern does

Kern identifies and inspects the fonts and typography used on web pages you choose to inspect. To do that, it reads the styling (fonts, sizes, colors, CSS rules) of the page in your current tab — locally, on your device — and displays that information to you. Page content is processed in memory and never leaves your browser.

What's stored on your device

Kern keeps a small amount of data in your browser's local extension storage (chrome.storage.local), purely so the tool works the way you left it:

  • Preferences — your theme (light/dark) and preferred color format (HEX/RGB/HSL/OKLCH).
  • Your collection — fonts you deliberately save, including their inspected properties.
  • Your last page audit — so the results are still there when you reopen the panel.
  • A first-run flag — so the welcome tip only shows once.

This data never leaves your device. It is removed by Chrome when you uninstall the extension, and you can clear it at any time from chrome://extensions.

Fonts you upload for try-on

The try-on feature lets you preview a font file from your own computer on a live page. Your file is read into memory in the current tab only. It is never uploaded anywhere, never written to storage, and never persisted — it's gone when the page reloads.

Network requests

Kern makes exactly one kind of network request, and only when you ask it to: when you use try-on, pairing previews or glyph coverage with a Google Fonts family, Kern loads that font's stylesheet and font files from Google Fonts (fonts.googleapis.com / fonts.gstatic.com), exactly as any website using Google Fonts does. This request is initiated by your action, contains no personal data from Kern, and is subject to Google's privacy policy. Kern's own UI fonts are bundled inside the extension and load from your disk.

Why Kern asks for the permissions it does

  • Read data on all websites — Kern must read a page's fonts and CSS to identify them, on whatever site you happen to be inspecting, without a per-site prompt each time. All processing is local; nothing is sent anywhere.
  • Scripting / active tab — injects the on-page inspector UI into the tab you're using.
  • Storage — the local preferences and collection described above.
  • Context menus, side panel, keyboard commands — the right-click entry, the workspace panel, and the toggle shortcut.

Third-party services

Kern's pages link to Ko-fi (optional donations) and niallcodes.com. Those sites are separate services with their own privacy policies; Kern shares no data with them — following a link is the only interaction.

Children

Kern does not knowingly collect information from anyone, including children. There is no data collection mechanism in the product.

Changes to this policy

If a future update ever changes how Kern handles data (for example, adding an opt-in feature that requires a network service), this policy will be updated first, the change will be described in the extension's changelog, and the Chrome Web Store data disclosures will be updated to match before release.
